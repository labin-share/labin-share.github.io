    1. Spring cloud config

    项目中有时候需要下面的需求，那么可以利用spring cloud config功能来实现。

    1. 集中管理配置。修改一个配置，各个服务都能生效
    2. 运行期间动态的调整配置

    它的原理是，将配置文件集中放在一个地方，比如git仓库。然后搭建一个config server，作用是从git中拉取配置。各个微服务通过访问config server获取最新的配置信息

    流程如下：

    1. 各个微服务访问config server，获取自己想要的配置

    2. config server去到git 仓库，拿去对应的配置文件，保存一份到本地，然后返回给微服务。

    3. 微服务会保存这个配置到内存中，之后微服务就读取内存的值，不会再去到config server。

    4. 如果需要刷新微服务中的配置，微服务会再次请求config server，config server会从git拉取最新的值。

    6.2 自动刷新

    自动刷新是指，当配置文件改变之后，config server会通知各个微服务拉取最新配置。

    流程：

    1. 配置改变之后，手动刷新config server。（通过手动调用config server的/bus/refresh接口进行刷新）
    2. config server拉取最新数据之后，将通知消息发送到rabbimq
    3. config client会监听rabbitmq，当有新的消息时，就会去请求config server，拉取新的配置。

    ==一旦有消息，所有的config client都会重新去请求config server吗==

    6.3 手动刷新和自动刷新的区别

    1. 手动刷新是要调用每个微服务的刷新接口进行刷新
    2. 自动刷新指直接刷新config server，config server就会通知所有的config client去刷新
